Key Technical Points

01

OAuth2 Authorization Code Flow 직접 구현

Spring Security 없이 순수 Servlet 환경에서 네이버·카카오 소셜 로그인을 직접 구현했습니다. socialId 기반 DB 조회로 기존/신규 회원을 자동 분기 처리하고, 신규 회원은 추가 정보 입력 후 정식 회원으로 전환되는 흐름을 설계했습니다. CSRF 방어를 위한 state 파라미터도 적용했습니다.

02

배송지 스냅샷으로 주문 이력 정합성 유지

배송지를 FK 참조 방식으로 관리하면 회원이 주소를 수정했을 때 과거 주문 이력의 배송지도 바뀌는 문제가 발생합니다. 이를 해결하기 위해 주문 시점에 배송지 정보를 주문 테이블에 직접 복사해 저장(스냅샷)하는 방식을 적용했습니다. 도메인 특성상 주문 시점 데이터 보존을 우선해 역정규화를 의도적으로 적용하였습니다.

03

주문 상태 3단계 분리로 재고 차감 시점 제어

초기 설계에서는 관리자가 승인 클릭 시 완료 처리와 재고 감소가 즉시 발생하여, 잘못 승인해도 되돌리기 어려운 구조였습니다. 이를 개선하기 위해 REQUEST → APPROVED(처리대기) → COMPLETED 3단계로 상태를 분리하여 승인과 재고 차감 시점을 분리했습니다. 관리자 실수로 인한 즉각적인 재고 변경을 차단했습니다.

04

PPT 문서화 — 스토리보드 · 시퀀스 다이어그램 · 트러블슈팅

설계 단계부터 기능 명세, 시퀀스 다이어그램, 트러블슈팅까지 전 과정을 문서화했습니다. 소셜 로그인 흐름, 주문 상태 전이, 배송지 스냅샷 설계 결정 근거를 포함하였습니다. 팀원 간 기능 이해와 커뮤니케이션이 원활하도록 협업 중심의 문서화 환경을 구축했습니다.

05

수동 배포 — Vultr VPS + Tomcat + Nginx

CI/CD 자동화 이전에 서버 구조를 직접 이해하기 위해 수동 배포를 선택했습니다. Vultr VPS에 Tomcat · Nginx · Oracle XE(Docker)를 직접 설치하고 도메인 연결까지 진행하며 서버 구성 전반을 경험했습니다. 이후 CIEL 프로젝트에서 Jenkins + Docker 자동화 배포로 발전시켰습니다.
